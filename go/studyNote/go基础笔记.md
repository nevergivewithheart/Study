# Go基础学习笔记
## 一、初识Go
### 1.1 Go的特点与优势
- 简单易学
- 自由高效
  - 拥有接近c语言的运行效率与接近PHP的开发效率
  - 支持当前所有的编程范式
    - 过程式编程
    - 面向对象编程
    - 面向接口编程
    - 函数式编程
- 强大的标准库
  - Go的标准库非常稳定
  - 标准库覆盖：网络、系统、加密、编码、图形等各个方面
  - 网络和系统库非常实用
- 部署方便
  - 不使用虚拟机
  - 可直接输出目标平台的二进制可执行文件
- 原生支持并发
  - 基于Goroutine，类似线程，但并非线程
  - 创建Goroutine的成本很低，只需要几千个字节的额外内存
  - Goroutine之间可以通过channel实现通信
  - Goroutine以及基于channel的并发性方法可最大限度地使用CPU资源
- 稳定性强
  - 拥有强大的编译检查、严格的编码规范
  - 提供软件生命周期的各个环节的工具，如go tool 、go fmt、go test
- 垃圾回收
  - 使用者只需要关注内存的申请而不必关心内存的释放
  - 内置runtime来进行自动管理

### 1.2 Go语言的结构组成及编码规范
#### 1.2.1  Go语言的结构组成
- 必须在源文件的非注释的第一行指定该文件属于哪个包
- package main 表示一个可独立执行的程序，每个go应用程序都需要包含一个名为main的包，并且该包包含一个叫main()的函数【该函数是go可执行程序的起点，既不能带参数，也不能定义返回值】
- 所有go函数以关键字func开头
- 每一个可执行程序都必须要包含main函数，通常是程序启动后第一个执行的函数，如果有initial()函数会先执行init()函数
- 只有package名称为main的包可以包含main函数
- 一个可执行程序有且仅有一个main包
- 通过import关键字来导入其他非main包
- 可以通过import关键字单个导入，也可以同时导入多个
